🍿 Movie Recommendation System & Watchlist Tracker ✅ DONE

A Netflix-style, fully automated Streamlit dashboard featuring Content-Based NLP recommendations, genre sorting, TMDB API fallbacks, and persistent user-state management with 1-10 rating systems.

1️⃣ System Overview & Objectives ✅ DONE

The goal of this project is to parse a raw CSV dataset of nearly 10,000 movies and transform it into an interactive, highly personalized streaming dashboard experience.

  • Dynamic Exploration: Filter movies by genre, release year, and popularity dynamically without reloading the page.
  • NLP Recommendations: Content-based filtering using TF-IDF text analysis on movie overviews.
  • State Automation & Rating: Persistent JSON databases track "Watched", "Currently Watching", or "My List". Users can rate interacted movies out of 10.
  • Immersive Details: Clicking a movie card loads a detailed view containing trailers, clips, and full metadata.
  • API Fallbacks: If the CSV is missing a Poster_Url, the system automatically falls back to fetching posters dynamically from TMDB/IMDB APIs.

API Fallbacks & Dynamic Filtering ✅ DONE

The system heavily relies on stunning visual posters. If a row has a blank or broken Poster_Url, the pipeline dynamically pings the TMDB API for movies or the TVMaze API for TV shows. To maintain a premium UI, a Dynamic Filtering Engine ensures that any media completely failing to retrieve a poster is automatically skipped and replaced in real-time before hitting the discovery grid.

3️⃣ Methods & Tech ✅ DONE

The tech stack leverages modern Python data science and frontend bridging tools:

  • Frontend: Streamlit for rapid UI prototyping, layout columns, and session state.
  • Data Manipulation: Pandas for high-performance CSV reading, null-value dropping, and string parsing (converting comma-separated genres into Python lists).
  • Machine Learning: Scikit-learn for the NLP pipeline.
  • External APIs: Requests module to fetch TMDB posters and YouTube trailer embed links.
  • State Persistence: JSON standard library to act as a lightweight local database for user watch history and ratings.

4️⃣ Algorithms Used ✅ DONE

The brain of the system uses Content-Based Filtering rather than Collaborative Filtering. This means it doesn't need thousands of other users to figure out what you like—it only needs to know the movies you interact with.

TF-IDF Vectorization ✅ DONE

(Term Frequency-Inverse Document Frequency)
The system mathematically converts the English text in the Overview column into numerical vectors, assigning higher importance to rare, descriptive plot words while ignoring common words like "the".

Cosine Similarity ✅ DONE

Calculates the angular distance between a requested movie's vector and all 9,848 other movie vectors. A score of 1.0 means identical plots; 0.0 means completely unrelated. The highest scores are returned as recommendations.

Weighted Popularity Sorting ✅ DONE

Pure text similarity isn't enough. We multiply the cosine score by a genre-overlap penalty and add a slight popularity boost (using Vote_Average and Popularity columns) to prevent recommending highly obscure, poorly-rated films.

6️⃣ Streamlit UI Blueprint & Functional Flow ✅ DONE

The frontend is divided into distinct tabs with deep-dive interactions.

📺 Tab 1: Discover (The Netflix View)

Features horizontally scrolling rows of posters: Trending Now, Top Rated, New Releases, and Because you watched [X].

🔍 Tab 2: Search & Filter

Power-user dashboard containing st.multiselect for Genres, st.slider for Release Date, and text search. Results update instantly.

📚 Tab 3: My Library & Ratings

Shows all movies marked as "My List", "Currently Watching", or "Watched". Users can update the status or assign a Rating (1 to 10 stars) to any movie in their library.

🎬 Clickable Cards & Stateful Dialogs ✅ DONE

Instead of clumsy button rows or page-reloading URL parameters, the app utilizes highly customized UI hacks and modern Streamlit features:

  • Invisible CSS Overlays: Entire poster cards act as clickable zones using advanced :has() CSS selectors that stretch an invisible Streamlit button over the HTML layout.
  • Stateful Pop-ups: Clicking a card triggers an @st.dialog fragment. This opens the movie details without causing a full-page refresh, maintaining scroll position and context.
  • Safe Action Execution: Background actions like "Add to My List" or "Rate 1-10" avoid fragile on_click Streamlit callbacks and instead use robust procedural logic paired with st.rerun().
